検査ミニポータル 一式(2026-07-16 時点・オフライン対応版)
====================================

GitHub Pages 用のフォルダ構成そのままです。
リポジトリ直下に kensa-tools/ の中身を置いてください。

・index.html … メニュー(検索・お気に入り付き)
・各フォルダ/index.html … 各ツール(全30種)
・sw.js / manifest.json / icon-*.png … オフライン(PWA)用

全ツールとも外部通信なし・端末内で計算のみ。

■ オフライン専用機としての使い方
1. 専用にするスマホで、一度だけオンラインでトップページを開く。
   → 「オフライン準備が完了しました(67件を保存)」と出れば、全30ツールが端末に保存済み。
2. トップページ下の「オフライン診断」を押し、「67 / 67 件を保存済み」を確認する。
   → 数が足りない場合は未保存のファイル名が出ます。電波の良い場所で開き直してください。
3. ブラウザメニューの「ホーム画面に追加」でアイコンを作成。
   → 全画面アプリとして起動し、以後は機内モードでも全ツールが動きます。
4. 現場に出す前に、機内モードで数ツール開いて実地確認する。

■ ツールを更新したとき（作者向け）★重要
・sw.js の var CACHE = "kensa-tools-vN" の数字を上げる。
・一式を再アップロードし、専用機で一度オンラインで開く。
  → 「新しいバージョンを取得しました」と出て、次回起動時に最新へ入れ替わります。
・番号を上げ忘れると、専用機は古いキャッシュのままになります(オフライン優先のため)。

■ 数字入力の共通仕様(2026-07-15 導入 / 全29ツール共通)
・全入力欄が type="text" + inputmode で、OSの数字キーパッド(電卓配列)が出ます。
  かな入力への切り替えは不要です。
・全角数字(１２３)・句点(。)・長音(ー)は自動で半角の 123 . - に直します。
・かな・英字・記号は自動で除去されます。
・入力欄をタップすると、右上に小さな補助ボタンが出ます。
    ±  … 数字キーパッドに無いマイナスを付ける/外す(公差の下限、座標、傾きなど)
    ✕  … その欄を一気に消す
  整数だけの欄(穴数・歯数など)には ± は出ません。
・「5 43」のような度分秒のスペース区切り入力はそのまま通ります
  (taper / right-triangle / hole-builder / multi-hole / phase-shift /
   pin-wire / bend-guide / bend-guide-3d が該当)。
・実装は各ツール末尾の共通スニペット。DOM構造は変えないため、
  表を再描画するツール(pitch-row 等)でも壊れません。

2026-07-17  TOOL-30 穴配置 総合 を新規追加                        CACHE=v9
    PCD・単独の穴・軸配置・端面基準を、ひとつの共通原点の上で合成する。
    図面まるごとを1枚で入力し、全穴の座標と総当たりピッチを出す。

  ■ 部品は4種類。ひとつの原点 (0,0) の上に好きに重ねられる（最大8個・60穴）
    ・PCD 等配   … PCD/中心X,Y/穴数/起点角/回り方向
    ・PCD 不等配 … 隣との角度（合計360）または真上からの絶対角
    ・任意穴     … 「X,Y で」「角度と距離で」「端面から」の3通り
    ・軸配置     … 基点＋中心線の角度＋距離L＋先端から直角に左右へ振る

  ■ 軸配置を入れた理由（TOOL-22 でもできたが、暗算が必要だった）
    「原点から45°に50mm、その先端から直角に左右15」を TOOL-22 で出すには
    θ=135 と θ=-45 を頭で計算して入れる必要があった（45+90 と 45-90）。
    図面には「45」「50」「15」「15」としか書いていないのに、打ち込む数字が
    違う。しかも間違えてもそれらしい座標が出るので気づけない。
    TOOL-30 は図面の数字をそのまま入れれば出る。
    左右は「基点から先端を見たときの左手・右手」で定義（符号を使わない）。

  ■ 角度規約は 真上0°・時計回り＋（TOOL-02 / TOOL-29 と同じ）★重要
    TOOL-22 穴位置ビルダーは X軸0°・反時計回り＋ で、これと逆（y=x の鏡像）。
    ちょうど45°だと両方とも同じ点になるため違いが隠れる。30°なら
    (25, 43.3) と (43.3, 25) で全く別の場所になる。TOOL-22 のフッターに
    規約の違いを明記した。

  ■ 端面基準は「ワーク外形」に依存する
    W・H と「共通原点の位置（中心 / 左下隅）」を入れる。中心と左下隅を
    取り違えると、端面基準で入れた穴だけが W/2・H/2 ずれる。図で確認のこと。

  ■ 検証
    ・PCD の計算が TOOL-29 と完全一致することを実測（23穴・253組、差 0.00e+0）。
      偏心・反時計回り・不等配を含む4円で照合。PCDのロジックは TOOL-29 と同一。
    ・「45/50/15/15」だけの入力で、手計算の理論値と小数3桁まで一致。2穴間 30.000。
    ・4種類の部品を混在（8部品・27穴・351組）させ、任意穴XY・極座標・端面基準・
      軸配置を個別に検算。原点5通りで全351組のピッチが不変。
    ・数字入力の共通スニペットは既存ツールと1文字も違わないことを照合済み。

  ■ 相互案内
    ・TOOL-29 → PCD以外が混ざるなら TOOL-30 へ。
    ・TOOL-22 → PCDが絡む/左右振り分けなら TOOL-30 へ（角度規約の違いを明記）。
    ※ TOOL-29 は据え置き。TOOL-30 が現場で認められた時点で、畳むかを判断する。
      TOOL-02（弦長の一発電卓）は速さが価値なので今後も据え置き。
